# 데이터 전처리 방식에 따른 성능

> **머신러닝 모델(Perceptron, SVM, Random Forest 등)이 동일한 데이터셋에서 전처리 방식(정규화, 표준화, 결측치 처리)에 따라 성능이 어떻게 달라질 수 있는지 사례를 들어 설명하시오.**


## 데이터 전처리란
- 데이터 분석을 위해 수집한 데이터를 분석에 적합한 형태로 가공하는 과정
- 불필요 데이터 제거나 이상치를 처리하여 데이터의 질을 향상 시킬 수 있음

데이터 전처리 방법 리스트
- 결측치 처리
- 이상치 처리
- 데이터 정규화
- 데이터 표준화
- 데이터 분포 변환
- 데이터 단위 변환
- 데이터 병합
- 데이터 분할
- 데이터 그룹화
- 데이터 변환 함수
- 데이터 피벗

그 중 살펴볼 데이터 전처리 방법
- 결측치 처리
- 정규화
- 표준화
- 이상치 처리
- 데이터 증강

### 결측치 처리
- NA/NULL 포함 행 혹은 일부 행을 제거하여 값이 없는 데이터를 삭제해 데이터의 값은 보존하기 위해 사용된다.
- `fillna()`를 통해서 Null값을 채우거나 `dropna()`로 값을 지우는 형식

### 정규화
- 전체 데이터의 값을 0~1사이의 값으로 치환한다. (스케일링)
- Min-Max Scaling
    - 모든 피처가 반드시 0과 1 사이의 값에 위치하도록 데이터를 변환한다. 그럼 2차원일 경우? 모든 데이터가 xy축의 0과 1 사이의 사각 영역에 속해진다.
- 수식: `(x - min) / (max - min)`
- **단점**: 이상치(outlier)에 매우 민감하다. 극단적인 값이 하나만 있어도 나머지 데이터가 좁은 범위에 몰린다.

### 표준화
- 데이터의 평균을 0, 분산 및 표준편차를 1로 만든다.
- 서로 다른 데이터들을 비교하기 용이하다.
- z-score을 자주 이용하며 z-score는 해당 데이터가 평균으로부터 표준편차의 몇배 떨어져있는 지 판단 가능하다.
- 요약: 데이터 표준화란 데이터 값들을 정규분포 형식의 값으로 변형한다.
- 수식: `(x - mean) / std`
- **정규화와 차이**: 이상치에 상대적으로 덜 민감하다. 결과 범위가 0~1로 고정되지 않는다.

### 이상치 처리
- 데이터 분포에서 크게 벗어난 값(이상치)을 제거하거나 대체하는 과정
- **IQR(Interquartile Range) 방식**
    - Q1(25%)과 Q3(75%) 사이의 범위를 기준으로 이상치를 판정
    - `Q1 - 1.5 * IQR` 미만 또는 `Q3 + 1.5 * IQR` 초과하는 값을 이상치로 처리
- **주의**: 전체 데이터 기준으로 IQR을 적용하면 소수 클래스(샘플 수가 적은 클래스)가 이상치로 판정되어 전부 삭제될 수 있다. 이 경우 **클래스별 IQR**을 적용해야 한다.

### 데이터 증강 (SMOTE)
- SMOTE(Synthetic Minority Over-sampling Technique): 소수 클래스 샘플을 합성하여 클래스 불균형을 해소하는 기법
- 기존 소수 클래스 샘플들 사이를 **선형 보간**하여 새로운 합성 샘플을 생성
- **주의사항**
    - SMOTE는 반드시 **훈련 데이터에만** 적용해야 한다 (테스트 데이터에 적용 시 data leakage 발생)
    - 소수 클래스 샘플이 극히 적으면 (예: 5개 미만) 합성 샘플이 직선 형태로 생성된다. `k_neighbors` 값을 줄여서 완화할 수 있다.
    - SMOTE 적용 후 **전체 정확도가 오히려 낮아질 수 있다**. 이는 정확도(Accuracy)가 불균형 데이터에 적합하지 않은 지표이기 때문이다. SMOTE는 소수 클래스의 recall을 높이지만 다수 클래스 경계를 흔들어 전체 accuracy가 하락할 수 있다.

**기본 전처리 정확도**

| 전처리 방식 | 정확도 |
|--|--|
| 데이터 전처리 전 | Failed (문자열 컬럼으로 인해 에러 발생) |
| 결측치 제거 | 0.8667 |
| 정규화 (MinMaxScaler) | 0.8000 |
| 표준화 (StandardScaler) | 0.8889 |

> 정규화가 결측치 제거보다 낮은 이유: PUBG 데이터 특성상 상위 플레이어(이상치)가 존재하여 MinMaxScaler가 나머지 데이터를 좁은 범위에 몰리게 만들기 때문이다.

---

**이상치 제거 후 정확도** (클래스별 IQR 적용)

| 전처리 방식 | 정확도 |
|--|--|
| 이상치 제거 + 정규화 | 0.9737 |
| 이상치 제거 + 표준화 | 1.0000 |

> 전체 기준 IQR을 적용하면 Silver 클래스(7개)가 전부 이상치로 판정되어 삭제되므로, 클래스별 IQR을 적용하였다.

---

**데이터 증강 (SMOTE) 정확도**

| 전처리 방식 | 정확도 |
|--|--|
| SMOTE + 표준화 | 이상치 제거 + 표준화보다 낮음 |
| 이상치 제거(클래스별) + SMOTE + 표준화 | example.py 실행 참조 |

> SMOTE 적용 후 accuracy가 낮아지는 현상은 정상이다. Accuracy는 불균형 데이터에 적합하지 않은 지표로, 실제로는 Silver 클래스의 recall이 개선되었을 수 있다. 정밀한 평가를 위해서는 F1-score 또는 Confusion Matrix를 확인해야 한다.
