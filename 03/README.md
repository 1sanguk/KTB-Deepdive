## 1. Python List vs NumPy 배열 — 메모리 구조

### Python List

```python
l = [1, 2, 3, 4, 5]
a = ['안녕', 12, '놀자']
```

- 동일한 원소로만 이루어질 수도 있고, 다양한 타입의 원소로도 이루어질 수 있음
- 내부적으로 **Dynamic Array**로 구현 — 연속된 메모리 공간에 **객체의 주소값**을 저장

```
[1, 2, 3, 4, 5] 선언 시 메모리 구조:

각 객체(1, 2, 3, 4, 5)가 메모리 어딘가에 개별 저장
        ↓
list는 이 객체들의 주소값을 담는 배열로 동작

[ptr→1] [ptr→2] [ptr→3] [ptr→4] [ptr→5]
```

- 메모리 부족을 대비해 실제 선언 크기보다 **약 1.125배** 여유 capacity 확보
- 주소값 오버헤드로 인해 **메모리 소비가 큼**

### NumPy ndarray

파이썬의 편리함 + C언어의 속도를 결합한 구조.  
객체 인터페이스는 Python, 메모리 관리는 C언어 스타일.

**ndarray 내부 구성**

| 구성 요소 | 설명 |
|-----------|------|
| **메타데이터** | `dtype`, `shape`, `strides`, `ndim`, `size`, `byteorder`, `data(버퍼 포인터)` |
| **데이터 버퍼** | 순수 데이터로 채워진 C언어 스타일 연속 배열 |

```
데이터 버퍼 구조:
[1][2][3][4][5][6]  ← 메모리에 빈틈없이 연속 배치
```

## 2. NumPy 배열 생성 과정

`np.array([[1,2,3], [4,5,6]])` 실행 시 내부 흐름:

### Step 1. 타입 추론 및 단일화

- Python 리스트 안의 요소 타입을 먼저 추론
- 정수와 실수가 섞여 있다면 더 큰 타입(`float64`)으로 **업캐스팅**

### Step 2. C 배열 메모리 할당

```
전체 원소 6개 × int64(8 bytes) = 48 bytes
→ C언어 malloc()으로 48 bytes 물리 메모리 한 번에 확보
```

Python list처럼 각 원소의 주소값을 따로 저장하지 않음.

### Step 3. 데이터 복사 및 패킹

- 확보한 연속 메모리에 **순수 원시 데이터만** 저장 (객체 오버헤드 제거)

### Step 4. 메타데이터 기록

- `strides`로 다차원 인식 정보 저장

```
2행 3열 배열의 strides = (24, 8)
  → 다음 행으로 이동: 24 bytes 점프
  → 다음 열로 이동:   8 bytes 점프
```

## 3. 연산 — 벡터화 (Vectorization)

### Python List 연산

```python
a = [1, 2, 3]
b = [4, 5, 6]
c = a + b   # [1, 2, 3, 4, 5, 6]  ← 이어 붙이기
d = c * 2   # [1, 2, 3, 4, 5, 6, 1, 2, 3, 4, 5, 6]  ← 반복
```

`+`는 리스트 연결, `*`는 리스트 반복 — 수학적 연산이 아님.

### NumPy 배열 연산

```python
a = np.array([1, 2, 3])
b = np.array([4, 5, 6])
c = a + b   # [5, 7, 9]   ← 원소별 덧셈
d = c * 2   # [10, 14, 18]  ← 원소별 곱셈
```

### 왜 NumPy가 훨씬 빠른가?

**Python 방식 (느림)**

```python
result = []
for i in range(len(a)):
    if type(a[i]) == type(b[i]):   # 매 원소마다 타입 체크
        result.append(a[i] + b[i]) # 매 원소마다 객체 생성
```

- 루프마다 타입 체크 + 객체 생성 반복 → 오버헤드 누적

**NumPy 방식 (빠름) — 벡터화**

```python
c = a + b          # 내부적으로 np.add(a, b) 실행 (Universal Function)

# 메모리 재사용 버전 (GC 부담 감소)
c = np.zeros(3)
np.add(a, b, out=c)
```

```
a: |1|2|3|
b: |4|5|6|
        ↓ SIMD 명령어로 한번에 처리
c: |5|7|9|
```

- 타입이 이미 확정 → **타입 체크 1회**만 수행
- 연속 메모리 배치 → **CPU 캐시 히트율 높음**
- CPU의 **SIMD** (Single Instruction Multiple Data) 명령어로 여러 블록을 동시 처리

## 4. 슬라이싱 비교

예시: 1,000만 개 데이터 짝수 인덱스 추출

### Python List 슬라이싱 — 데이터 복사

```python
list_a = [i for i in range(10_000_000)]
list_b = list_a[::2]  # 500만개 새 리스트 생성
```

- 500만 개 짜리 새 리스트 객체 + 주소값 저장 공간 생성
- 메모리 확보 + 할당 시간 소요
- **독립성 보장**: `list_b[0] = 9999` 해도 `list_a[0]` 불변

### NumPy 슬라이싱 — 메타데이터만 수정

```python
arr_a = np.array([i for i in range(10_000_000)])
arr_b = arr_a[::2]   # 메타데이터만 변경
```

```
기존 strides: (8,)   → 8 bytes씩 이동
변경 strides: (16,)  → 16 bytes씩 이동 (한 칸 건너뜀)
```

- 실제 데이터 복사 없음 → **매우 빠름**
- **독립성 없음**: `arr_a`와 `arr_b`는 동일한 메모리 공유

## 5. 브로드캐스팅 (Broadcasting)

shape이 다른 배열끼리 연산할 때, **작은 배열을 확장**하여 계산하는 메커니즘.  
실제 메모리 복사는 일어나지 않음 — 계산 시점에만 확장된 것처럼 처리.

### 브로드캐스팅 규칙

> 차원 비교는 **오른쪽 정렬**부터 시작

| 규칙 | 내용 |
|------|------|
| **규칙 1** | 두 배열의 해당 차원 크기가 같으면 연산 가능 |
| **규칙 2** | 같지 않다면, 둘 중 하나가 반드시 **1** 이어야 함 |

```
(4, 3, 2) + (2,) 계산 과정:

  (4, 3, 2)
+       (2)   ← 오른쪽 정렬
-----------
  (4, 3, 2)
+ (1, 1, 2)   ← 빈 차원을 1로 채움
-----------
= (4, 3, 2)   ✅
```

### 케이스 1: 열벡터 + 행벡터

```python
col = np.array([[1],
                [2],
                [3]])      # shape (3, 1)

row = np.array([10, 20, 30])  # shape (3,) → (1, 3)

print(col + row)
# [[11, 21, 31],
#  [12, 22, 32],
#  [13, 23, 33]]
```

```
(3, 1) + (1, 3) → (3, 3)  ← 양쪽 모두 확장되는 케이스
```

### 케이스 2: 게임 서버 버프 적용

```python
# 플레이어 100명의 스탯 [공격력, 마나, 체력]
players = np.array([[100, 50, 200],
                    [80,  60, 180],
                    ...])   # shape (100, 3)

# 이벤트 버프: 공격력+10, 마나+5, 체력+50
buff = np.array([10, 5, 50])  # shape (3,)

# 브로드캐스팅으로 100명 한 번에 적용
buffed = players + buff  # (100, 3) + (3,) → (100, 3)
# buff가 [[10, 5, 50], [10, 5, 50], ...] 처럼 동작 (메모리 복사 없이)

# 결과:
# [[110, 55, 250],
#  [90,  65, 230], ...]
```

### 케이스 3: 브로드캐스팅 실패

```python
a = np.array([1, 2, 3])  # shape (3,)
b = np.array([1, 2])     # shape (2,)
a + b   # ValueError ❌
```

```
규칙 1: 3 ≠ 2  → 불일치
규칙 2: 3과 2 모두 1이 아님
→ 브로드캐스팅 불가
```

해결 방법:

```python
a = np.array([1, 2, 3])  # shape (3,)
b = np.array([1])         # shape (1,)
a + b  # [2, 3, 4]  ✅  ← b가 1이므로 브로드캐스팅 성공
```

## 핵심 요약

| 항목 | Python List | NumPy ndarray |
|------|-------------|---------------|
| 메모리 구조 | 객체 주소값 배열 | 연속 원시 데이터 버퍼 |
| 타입 | 혼합 가능 | 단일 타입 (업캐스팅) |
| 연산 방식 | 인터프리터 루프 | SIMD 벡터화 (C레벨) |
| 슬라이싱 | 실제 데이터 복사 | 메타데이터(strides)만 변경 |
| 독립성 | 슬라이스 독립 | 뷰(View) 공유 |
| 브로드캐스팅 | 지원 안 함 | 지원 (메모리 복사 없음) |
